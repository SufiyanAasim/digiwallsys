## 📖 Overview

**Current** establishes the primary verified flow of funds into digiwallsys.

## 🎯 Objectives

- Remove generated balance credits from product flows.
- Create provider checkout intents without trusting the client.
- Credit funds only after a signed, deduplicated provider event.
- Surface funding state and failures clearly.

## ✨ Planned scope

### 🏦 Verified funding

- Provider intent and reference lifecycle.
- HMAC webhook verification and event replay protection.
- Clearing-account-to-wallet ledger journals.
- Funding history and notifications.

## 🏛️ Architecture progress

The provider is isolated behind checkout and webhook configuration. Funding is
posted transactionally only after the external trust boundary verifies it.

## 💻 Compatibility

Node.js 20+ · Expo SDK 53 · PostgreSQL 14+ · HTTPS funding provider
