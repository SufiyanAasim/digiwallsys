> *Name: Voyage*

**Status:** Completed

---

## 📖 Overview

**Voyage** expands payment access through QR interaction, biometrics, and
broader-market foundations.

## 🎯 Objectives

- Generate and scan secure payment-request QR payloads.
- Require device biometrics before unlocking a saved session.
- Establish currency-aware accounts and regional formatting.
- Improve localization and accessibility coverage.

## ✨ Planned scope

### 🌊 Broader access

- Camera-based QR payment request scanning.
- SecureStore-backed biometric session unlock.
- Currency-aware wallet and ledger account schema.
- Localization, formatting, and accessibility acceptance gates.

## 🏛️ Architecture progress

QR codes carry only an opaque request UUID; the authenticated API resolves and
authorizes the current request before any payment occurs.

## 💻 Compatibility

Android/iOS camera · device biometrics · Expo SDK 53
