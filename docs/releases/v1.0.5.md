> *Name: Drift*

**Status:** Planned

## 📖 Overview

**Drift** refines the Anchor identity flow with recoverable, verifiable, and
device-aware sessions.

## 🎯 Objectives

- Harden refresh-token rotation and revocation behavior.
- Complete email verification and password recovery delivery.
- Validate biometric unlock across supported devices.
- Refine authentication errors from Anchor feedback.

## ✨ Planned scope

### 🔐 Session lifecycle

- Rotating hashed refresh tokens with replay response.
- Password reset, account lockout recovery, and email verification.
- SecureStore-backed mobile sessions and biometric unlock.

## 🏛️ Architecture progress

Authentication moves from a single access token to a short-lived access token
plus server-tracked rotating refresh sessions and queued delivery adapters.

## 💻 Compatibility

Node.js 20+ · Expo SDK 53 · PostgreSQL 14+
