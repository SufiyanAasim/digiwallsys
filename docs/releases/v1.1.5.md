## 📖 Overview

**Swell** prepares money movement for higher request and transaction volume.

## 🎯 Objectives

- Make repeated client requests safe.
- Bound abusive authentication and API traffic.
- Prove concurrent transfers cannot overdraw a wallet.
- Expose pool, query, and transfer-failure signals.

## ✨ Planned scope

### 📈 Scale controls

- Idempotency reservations and stored replay responses.
- API and authentication rate limits.
- Stable wallet lock ordering and concurrent-transfer integration tests.
- Load, soak, and database-pool monitoring gates.

## 🏛️ Architecture progress

Write requests receive a scoped idempotency record before business processing;
money movement remains serialized only for the wallets it modifies.

## 💻 Compatibility

Node.js 20+ · PostgreSQL 14+
