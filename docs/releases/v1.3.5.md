## 📖 Overview

**Beacon** adds visibility, reporting, receipts, and proactive user alerts.

## 🎯 Objectives

- Search and filter transaction history efficiently.
- Export safe CSV data and generate reference-addressed receipts.
- Notify users about money movement and security events.
- Trigger configurable spending-threshold alerts.

## ✨ Planned scope

### 🔎 Reporting and alerts

- Search, cursor pagination, direction/date/amount filters, and CSV export.
- Transaction receipts with stable UUID references.
- In-app inbox, Expo push delivery, and notification preferences.
- Spending, balance, funding, and schedule-failure alerts.

## 🏛️ Architecture progress

Notifications are persisted first and dispatched asynchronously, keeping payment
commits independent from push-provider availability.

## 💻 Compatibility

Expo SDK 53 · Expo push project · PostgreSQL 14+
